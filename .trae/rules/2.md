# Cline 全局规则

## 语言
- 始终用中文回复

## 工作流程（必须遵守）

### 收到任务后：
1. **分析** → 读取相关源码，找到根因（从代码中找证据，不要猜），根因必须穷举，每次你为已经穷举完成后，要重新检查是否穷举完成。
2. **规划** → 输出完整计划，等用户确认后再动手
3. **执行** → 按计划逐步实施
4. **验证** → 按验收标准逐条检查，构建测试通过后再报告

### 规划输出格式：
```
## 目标
（一句话说清楚要做什么）

## 根因
（从代码中找到的根本原因，附文件路径和行号）

## 实施步骤
1. 改文件 A：具体改什么，为什么
2. 改文件 B：具体改什么，为什么

## 验证标准
（可验证的标准）
```

### 验证完成后：
```
✅ 验收结果：
- [x] 标准1 已通过
- [x] 标准2 已通过
- [ ] 标准3 未通过（原因：...）
```

## 编辑原则
- 只修改需要改变的部分，不重写整个文件
- 用 edit 工具精确修改（old_string + new_string）
- 修改前先 read 相关文件理解上下文
- 遵循项目现有代码风格

## 约束
- 有疑问先问再动手
- 不要执行破坏性操作
- 不要泄露私人信息
