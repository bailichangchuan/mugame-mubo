## 🗂️ 项目规则加载顺序

当用户打开一个项目时，规则按以下顺序加载（后面的覆盖前面的）：
1. **全局规则**（本文件）— 所有项目通用
2. **项目级规则**（`.trae/rules/*.md` 或 `.claude/skills/`）— 该项目专用

如果项目里有规则，以项目规则为准。全局规则只提供通用模板。

## 🎯 技能使用示例

| 用户说 | AI 做的 |
|--------|---------|
| "验证一下这个按钮能不能用" | 读取 `verify` 技能 → 构建 → 测试 → 报告 |
| "帮我 review 一下这个 PR" | 读取 `review` 技能 → 分析 → 输出结构化报告 |
| "有 bug，报错 Cannot read property" | 读取 `debug` 技能 → 定位根因 → 给出修复方案 |
| "我要提交这次改动" | 读取 `commit` 技能 → 先展示 message → 确认后提交 |
| "10 个文件都要加日志" | 读取 `batch` 技能 → 拆解为 10 个任务 → 并行执行 |
| "感觉代码有点乱" | 读取 `simplify` 技能 → 快速审查 → 指出问题 |

## 编辑原则
- 只修改需要改变的部分，不重写整个文件
- 用 edit 工具精确修改（old_string + new_string）
- 修改前先 read 相关文件理解上下文
- 遵循项目现有代码风格
